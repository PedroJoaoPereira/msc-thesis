Image rescaling é a técnica utilizada para criar uma nova versão de uma imagem de diferente dimensão.
https://en.wikipedia.org/wiki/Comparison_gallery_of_image_scaling_algorithms
https://clouard.users.greyc.fr/Pantheon/experiments/rescaling/index-en.html
http://tanbakuchi.com/posts/comparison-of-openv-interpolation-algorithms/

O aumento da dimensão de uma imagem é designado de upsampling e a sua redução é designada de downsampling.
Estas últimas operações não são opostas uma vez que realizar um upsampling seguido de downsampling, ou o recíproco,
não resulta na imagem original, pois estas operações implicam perdas dos dados originais da imagem.
Este processo implica um compromisso entre a eficiência e rapidez de um algoritmo, e a qualidade do resultado.

Os diferentes algoritmos de rescaling de uma imagem são:
- Nearest neighbor
http://tech-algorithm.com/articles/nearest-neighbor-image-scaling/
Segundo este algoritmo, no caso de aumentar a dimensão de uma imagem, são introduzidos novos pixéis na imagem resultante
em que a sua intensidade é igual à intensidade do pxel mais próximo na imagem original.
No caso de diminuir a dimensão de uma imagem, a intensidade dos pixéis da imagem resultante será igual a um dos pixéis da imagem original.

Esta técnica é de rápida execução mas introduz imagens de fraca qualidade, criando imagens pixelizadas no caso de upsampling e
removendo pequenas arestas no caso de downsampling. Uma vantagem deste algoritmo é não introduzir novas cores na imagem de resultado.

- Bilinear interpolation
https://stackoverflow.com/questions/26142288/resize-an-image-with-bilinear-interpolation-without-imresize
http://tech-algorithm.com/articles/bilinear-image-scaling/

Segundo este algoritmo os pixéis da imagem resultante são obtidos a partir da interpolação linear dos pixéis
originais vizinhos do pixel a calcular. O cálculo será realizado por duas interpolações lineares, interpolação bilinear.

- Bicubic
- Lanczos
